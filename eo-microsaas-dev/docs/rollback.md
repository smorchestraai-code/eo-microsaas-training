# Rollback runbook — eo-microsaas-dev

If `1.1.0` ships broken on a student's machine, here's how to pin them back to `1.0.0` in under 5 minutes.

## Symptom signals

Use this runbook if a student reports any of:
- `/eo-guide` returns garbage phase ("I don't know" + no next command) after handover-bridge clearly succeeded
- `/eo-plan` or `/eo-score` crashes with `_dev-progress.md` write errors
- `handover-bridge` aborts at Step 0 despite `~/.claude/CLAUDE.md` being present
- CI workflow generated by Step 5b is invalid YAML

## Rollback command (student side)

```bash
# Pin to last known-good version
claude plugin install eo-microsaas-dev@eo-microsaas-training --version 1.0.0

# Verify
claude plugin list | grep eo-microsaas-dev
# Expected: eo-microsaas-dev@1.0.0
```

After rollback, the student loses:
- `/eo-guide` and `/eo-status` commands
- Auto-seeded `_dev-progress.md` (existing file keeps working, commands just stop writing to it)
- Step 0 precondition + Step 5b CI gate + Step 6b tracker seed in `handover-bridge`

They keep everything else (8 commands + 7 skills + 5-Hat scoring).

## Reporting the incident

Open a GitHub issue at `smorchestraai-code/eo-microsaas-training`:
- Title: `SEV? — eo-microsaas-dev 1.1.0 — {short symptom}`
- Body: paste student's OS, `claude --version`, `node --version`, the failing command, and the full error.
- Tag with `rollback-used`.

## Fix-forward

1. Reproduce the bug on a scratch macOS / Windows machine (or WSL2).
2. Write a failing fixture under `eo-microsaas-dev/skills/eo-guide/fixtures/` that captures the phase mis-detection.
3. Fix the skill or template.
4. Bump to `1.1.1`, add a `### Fixed` block to CHANGELOG.md, cut a release.
5. Ask rolled-back students to upgrade: `claude plugin update eo-microsaas-dev@eo-microsaas-training`.

## Do not

- **Do not delete the `_dev-progress.md` the student built up** — even if it's corrupt. Copy it aside first: `cp _dev-progress.md _dev-progress.md.bak.$(date +%s)`.
- **Do not force a global re-install** — `install.sh` is for fresh setups. On existing projects, `claude plugin install … --version 1.0.0` is enough.
- **Do not push a hotfix to 1.1.0 in place.** Always bump the patch version (`1.1.1`), so students who already pinned to `1.0.0` see a signal to upgrade.
